About bugs

. Bugs Encountered and Resolved

a. Multiple Tk() Instances

  • Issue Faced: Initially, we created a new tk.Tk() instance for every dashboard (e.g., officer, admin, customer), thinking each new screen needed its own root window.
  • Problem: This caused inconsistent behavior and warnings. At times, windows wouldn’t display correctly or the application would hang.
  • How We Resolved It: We learned that only one Tk() instance should be active in a tkinter application. We changed all other windows to use tk.Toplevel() instead, keeping only one Tk() for the login window.

Invalid User Inputs (e.g., letters in amount fields)

  • Issue Faced: Users could input letters instead of numbers in fields like "Fine Amount" or "Payment Amount", which would crash the app.
  • How We Resolved It: We added input validation using .isdigit() checks and showed warnings if invalid input was detected.
